Node.js application 테스트용 embedded DB

2020-04-18
dockerode, javascript, test

Javascript 서버 개발을 하면서 기존의 Java 개발하던 지식과 경험을 많이 활용하는데, 이번에는 테스트를 할 때 개선점을 찾기 위해서 찾아본 바에 대한 것이다.

기존에 개발을 하면서 테스트 코드를 짜긴했지만 급하게 개발하다보니 테스트 환경에서 사용하는 DB는 로컬 DB를 사용하는 방식으로 했었는데,
매번 테스트 시 로컬에서 DB를 띄우는 방식이 번거롭기도 했고 또, 테스트는 어느 환경에서도 돌아가는 것이 좋기 때문에 개선이 분명 필요했다.

Java 같은 경우 테스트 시, H2나 embedded-redis와 같은 library를 통해 테스트 시 in-memory로 DB를 띄울 수 있다.
그래서 이와 비슷한 패키지가 node에서도 있지 않을까라고 생각해서 이와 관련해서 찾아본 것들을 정리하고자 한다.

예제 코드는 여기에서 확인 가능하다.

MySQL

MySQL과 같은 RDB는 Java에서 H2와 같이 runtime 시에 실행 가능한 embbeded DB를 찾질 못했다.
그래서 그 대신 Docker container를 사용하는 방식을 생각해봤다.

Dockerode를 사용하여 Docker repote API를 통해 MySQL container를 띄우고 내리는 코드는 아래와 같다.

container를 띄우면 코드 내에서 지정한 password로 root계정 접속이 가능하다.

Jest를 사용한다면 test setup file에 beforeAll에 시작하는 코드를 넣고 afterAll에 해당 container를 내리고 삭제하는 코드를 넣으면 될 것이다.

이 방식의 경우 Docker가 설치되어있어야 한다는 조건이 붙지만 이런 방식을 사용하지 않고 테스트용 DB를 띄우고 내릴 수 있는 다른 좋은 방법을 찾질 못하였다.
Java에서와 같이 H2를 런타임에 실행할 수 있는 편리한 패키지가 생긴다면 좋을 것 같다.

MongoDB

MongoDB는 mongodb-memory-server라는 embedded로 실행할 수 있는 패키지가 이미 존재한다.
앞서 살펴본 dockerode를 사용한 것과 다르게 npm 패키지를 다운받고 코드 몇 줄 추가하면 된다.

이 패키지의 좋은 점은 node 패키지이지만 MongoDB instance를 그대로 띄워준다는 것이다.
그렇기 때문에 Compass와 같은 client tool로도 접속이 가능하며,
설정을 통해 MongoDB 버전 지정, Replica Set 등도 가능하다.

github README에 예제 코드도 있고 설정 설명도 있기 때문에 쉽게 사용할 수 있을 것이다.

실행 후 얻은 connectionString를 통해 DB에 접속 가능하다.

Redis

Redis의 경우 mongodb-memory-server와는 다르게 DB instance를 띄워주는 것이 아닌,
node redis를 mocking해서 memory 상에서 Redis client의 command를 처리해주는 redis-mock 패키지가 있다.
node redis를 사용할 때 Redis client를 DI를 통해서 주입받는다고 할 때 주입받는 client를 redis-mock의 client를 주입받으면 쉽게 테스트가 가능할 것이다.

하지만 node redis 보다는 ioredis를 주로 사용할 텐데 ioredis도 동일하게 ioredis-mock 패키지가 존재한다.
ioredis-mock 같은 경우 일반적으로 사용되는 command의 경우 대부분 지원되지만 그래도 ioredis와 100% 호환되는 것은 아니기 때문에 해당 패키지를 사용할 때는 ioredis-mock에서 제공하는 비교표를 확인해보길 바란다.

ioredis-mock의 경우 ioredis 패키지가 필요하다.

하지만 개인적으로 DB를 통해 실제 connection을 사용하지 않고 mocking된 코드를 실행하는 것이기 때문에 선호하진 않는 방법이다.
차라리 MySQL과 동일하게 Docker로 Redis container를 띄우는 방식을 고려해도 나쁘진 않을 것 같다.
